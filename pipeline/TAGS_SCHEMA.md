# CDP Program Tag Schema

**Version:** 1.0 (2026-03-04)
**Column:** `tags` on `cdp_programs` table (TEXT, JSON)

## Overview

Every program in the CDP database is enriched with structured tags via the `enrich-tags.py` pipeline script. Tags are stored as a JSON object in the `tags` column and are used for filtering, personalized categories, and search.

## Tag Fields

### `career_stage` — Who can apply?
| Value | Meaning |
|-------|---------|
| `high_school` | Open to high school students |
| `undergraduate` | Open to undergraduate students |
| `graduate` | Open to graduate (master's) students |
| `phd` | Open to PhD students |
| `postdoc` | Open to postdoctoral researchers |
| `professional` | Open to working professionals |
| `any` | No academic stage restriction |

### `benefits` — What does the program provide?
| Value | Meaning |
|-------|---------|
| `stipend` | Financial stipend or pay |
| `housing` | Housing provided or housing allowance |
| `travel_funding` | Travel costs covered |
| `academic_credit` | Can receive academic credit |
| `health_insurance` | Health coverage included |
| `meals` | Meals or food stipend |
| `equipment_access` | Access to lab/computing equipment |

### `duration` — How long?
| Value | Meaning |
|-------|---------|
| `summer` | Summer program (general) |
| `8_weeks` | Approximately 8 weeks |
| `10_weeks` | Approximately 10 weeks |
| `12_weeks` | Approximately 12 weeks |
| `semester` | One academic semester |
| `year_round` | Year-round / ongoing |
| `part_time` | Part-time commitment |

### `location_type` — Where?
| Value | Meaning |
|-------|---------|
| `in_person` | In-person / on-site |
| `remote` | Fully remote |
| `hybrid` | Hybrid / flexible |

### `focus_type` — What kind of work?
| Value | Meaning |
|-------|---------|
| `wet_lab` | Hands-on laboratory research |
| `computational` | Computational / data / coding |
| `clinical` | Clinical or medical setting |
| `field_research` | Outdoor / field work |
| `policy` | Science policy or communications |
| `engineering` | Engineering / hardware / design |
| `industry` | Industry-style project work |
| `teaching` | Education / mentoring |
| `design` | Product / UX / graphic design |

### `special_eligibility` — Any special focus?
| Value | Meaning |
|-------|---------|
| `underrepresented_minorities` | Targeted at URM students |
| `first_generation` | First-generation college students |
| `women_in_stem` | Women-in-STEM focused |
| `veterans` | Military veterans |
| `disability` | Students with disabilities |
| `us_citizen_only` | US citizenship required |
| `open_international` | International students eligible |
| `need_based` | Financial need considered |

### `keywords` — Miscellaneous descriptors
| Value | Meaning |
|-------|---------|
| `paid` | Program includes pay (shorthand) |
| `prestigious` | Nationally competitive / well-known |
| `highly_competitive` | Very low acceptance rate |
| `beginner_friendly` | No prior research experience needed |
| `renewable` | Can be renewed for multiple years |
| `needs_recommendations` | Requires recommendation letters |
| `needs_transcript` | Requires academic transcript |
| `needs_gpa_3_0` | Minimum 3.0 GPA required |
| `federal_program` | US federal government program |
| `industry_partner` | Industry-sponsored or co-hosted |
| `research_intensive` | Heavy research component |

```json
  "benefits": ["stipend", "housing", "travel_funding"],
  "duration": ["summer", "10_weeks"],
  "location_type": ["in_person"],
  "focus_type": ["wet_lab", "computational"],
  "special_eligibility": ["us_citizen_only"],
  "keywords": ["paid", "prestigious", "needs_recommendations", "federal_program"]

```

## API Usage

### Filter by tag

```
GET /api/cdp/programs?career_stage=undergraduate&benefits=stipend&has_stipend=true
GET /api/cdp/programs?keywords=beginner_friendly&remote=true
GET /api/cdp/programs?special=underrepresented_minorities
GET /api/cdp/programs?focus_type=computational&field=cs
```

### Get available tag values
```
GET /api/cdp/programs/tags/summary
```
Returns all tag values currently in the DB, with counts of tagged programs.

### Combined search + filter
```
GET /api/cdp/programs?q=NIH&career_stage=undergraduate&benefits=stipend&page=1&limit=20
```

## Enrichment Process

Tags are generated by the `pipeline/enrich-tags.py` script:
1. Programs are processed in batches of 10
2. Claude (claude-haiku) reads each program's title, description, stipend, location, eligibility
3. Claude returns structured JSON tags
4. Tags are written to the `tags` column; `sector`/`categories` are backfilled if missing
5. `tags_enriched_at` timestamp is set

### Running enrichment
```bash
# Enrich all unenriched programs (single worker)
python3 pipeline/enrich-tags.py --unenriched-only

# Run full batch enrichment with 5 parallel workers
./pipeline/run-enrichment.sh

# Enrich a specific range
python3 pipeline/enrich-tags.py --offset 0 --limit 80

# Dry run to preview
python3 pipeline/enrich-tags.py --unenriched-only --dry-run
```

### Pipeline integration
`run-pipeline.sh` automatically runs `enrich-tags.py --unenriched-only` after each pipeline upsert, so new programs get tagged asynchronously.

## Sector Backfill

For the 182 programs loaded before the pipeline was established (IDs 232–413), the enrichment also infers and backfills:
- `sector` — from SECTOR_KEYWORDS matching on title + org + description
- `categories` — derived from sector (`["sector_name"]`)
